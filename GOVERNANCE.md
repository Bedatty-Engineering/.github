This document describes how decisions are made in **Bedatty Engineering** and how the shared defaults in this repository relate to individual projects. It is a **personal** organization, not a company or a formal foundation.

## Roles

- **Maintainer:** [@bedatty](https://github.com/bedatty) — prioritizes work, merges changes to org-wide defaults, and owns org settings.
- **Contributors:** anyone who opens issues or pull requests in good faith, under the [Code of Conduct](CODE_OF_CONDUCT.md).

## Decision-making

- **Day-to-day:** there is no steering committee or vote. The maintainer decides what to merge, when to respond, and how each repository evolves—see also [SUPPORT.md](SUPPORT.md) on timelines.
- **Changes to this repository** (templates, policies, org profile): propose via pull request; changes should stay consistent with [CONTRIBUTING.md](CONTRIBUTING.md) and existing community files.
- **Per-repository choices** (license, stack, whether PRs are welcome): live in that repository’s readme and issue tracker unless this org-wide documentation says otherwise.

## What this `.github` repository is

This repo holds **default community files** for the organization—issue forms, pull request templates, support and security policies, and the [organization profile](profile/README.md). Updates here shape how contributions are framed across the org; they do not replace readme-specific guidance in each project.
