This GitHub organization is for **personal projects** I publish and share. There is no separate “official” helpdesk or internal channel—everything goes through the public repo.

## When to use issues

Open an issue on the relevant repository for:

- reproducible bugs;
- small, concrete improvements;
- documentation fixes or gaps.

I look at issues **when I can**; there is no SLA.

## When *not* to use issues

Avoid issues for:

- general career or hiring questions (use the contact links on my profile if I list any);
- urgent production incidents in *your* systems—these repos are provided as-is;
- private or sensitive topics (use a private channel only if one is documented for that repo, e.g. security—see `SECURITY.md`).

## Choosing the right repo

Each project has its own issue tracker. Please open the issue in the repository that actually contains the code or docs you mean, so context stays in one place.
